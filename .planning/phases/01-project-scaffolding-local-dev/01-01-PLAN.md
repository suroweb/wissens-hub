**Step 1 - Convert each React component (.tsx) from class to functional:**

For each of the 4 web part React components (Dashboard.tsx, ArticleSidebar.tsx, Freigabecenter.tsx, AdminPanel.tsx):

1. Replace `class {Name} extends React.Component<IProps, {}>` with `const {Name}: React.FunctionComponent<IProps> = (props) => {`
2. Remove `constructor`, `componentDidMount`, and other lifecycle methods
3. Replace `this.props` with destructured `props`
4. Replace `public render()` method body with direct return from the function body
5. Add `export default {Name}` at the bottom
6. Keep the existing interface (IProps) -- just change how it's consumed

Each component should render minimal placeholder content:
- Component name as heading (e.g., `<h2>Dashboard</h2>`)
- A Fluent UI icon (use `Icon` from `@fluentui/react/lib/Icon`):
  - Dashboard: `ViewDashboard`
  - ArticleSidebar: `ReadingMode`
  - Freigabecenter: `Approve`  (Freigabecenter = Approval Center)
  - AdminPanel: `Settings`
- The `description` prop from property pane
- No feature code, no state management, no API calls

**CRITICAL:** The WebPart .ts files (DashboardWebPart.ts, etc.) MUST stay as classes extending `BaseClientSideWebPart`. Only the React .tsx components are converted to functional. The WebPart class renders the functional component via `React.createElement`.

**Step 2 - Handle the Application Customizer:**

The UnreadBadge Application Customizer (`UnreadBadgeApplicationCustomizer.ts`) stays as a class extending `BaseApplicationCustomizer` (SPFx requirement). No React component exists yet for this -- just the customizer shell with `onInit()` logging a console message. A React component for the badge UI will be added in Phase 8.
