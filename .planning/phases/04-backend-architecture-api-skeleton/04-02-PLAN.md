    **1. `api/src/WissensHub.Infrastructure/Services/CurrentUser.cs`**
    - Namespace: `WissensHub.Infrastructure.Services`
    - Using: `using System.Security.Claims;`, `using WissensHub.Application.Interfaces;`
    - Class: `public class CurrentUser : ICurrentUser`
    - Properties with private setters and safe defaults:
      - `public string UserId { get; private set; } = string.Empty;`
      - `public string DisplayName { get; private set; } = string.Empty;`
      - `public string Email { get; private set; } = string.Empty;`
      - `public IReadOnlyList<string> Groups { get; private set; } = [];`
    - Method `public bool IsInGroup(string groupName)`:
      - Return `Groups.Any(g => string.Equals(g, groupName, StringComparison.OrdinalIgnoreCase))`
    - Method `public void SetFromClaimsPrincipal(ClaimsPrincipal principal)`:
      - `UserId = principal.FindFirstValue("http://schemas.microsoft.com/identity/claims/objectidentifier") ?? principal.FindFirstValue("oid") ?? string.Empty;`
      - `DisplayName = principal.FindFirstValue("name") ?? string.Empty;`
      - `Email = principal.FindFirstValue("preferred_username") ?? string.Empty;`
      - `Groups = principal.FindAll("groups").Select(c => c.Value).ToList();`
    - Note: `FindFirstValue` is from `System.Security.Claims.ClaimsPrincipal` extension method. If not available, use `principal.FindFirst("name")?.Value ?? string.Empty` pattern instead.

    **2. `api/src/WissensHub.Functions/Middleware/UserIdentityMiddleware.cs`**
    - Namespace: `WissensHub.Functions.Middleware`
    - Using: `using System.Security.Claims;`, `using Microsoft.Azure.Functions.Worker;`, `using Microsoft.Azure.Functions.Worker.Middleware;`, `using Microsoft.Extensions.DependencyInjection;`, `using WissensHub.Application.Interfaces;`, `using WissensHub.Infrastructure.Services;`
    - Class: `internal sealed class UserIdentityMiddleware : IFunctionsWorkerMiddleware`
    - Method `public async Task Invoke(FunctionContext context, FunctionExecutionDelegate next)`:
      - Check if `context.Items.TryGetValue("User", out var userObj)` and `userObj is ClaimsPrincipal principal`
      - If yes, resolve `ICurrentUser` from `context.InstanceServices.GetRequiredService<ICurrentUser>()`
      - Cast to `CurrentUser` and call `SetFromClaimsPrincipal(principal)`
      - Always call `await next(context)` regardless of whether User was found (health endpoint has no User)
    - This middleware MUST be registered AFTER AuthenticationMiddleware in Program.cs (done in Plan 04)
