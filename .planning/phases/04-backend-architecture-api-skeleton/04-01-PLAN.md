  <action>
    Create 4 MediatR pipeline behaviors in `api/src/WissensHub.Application/Common/Behaviors/`. All use file-scoped namespaces, primary constructors, and implement `IPipelineBehavior<TRequest, TResponse> where TRequest : IRequest<TResponse>`.

    Registration order (outermost to innermost): Exception -> Logging -> Authorization -> Validation. This means Exception wraps everything, Logging wraps Authorization and Validation, etc.

    **1. `ExceptionBehavior.cs`**
    - Namespace: `WissensHub.Application.Common.Behaviors`
    - Primary constructor: `(ILogger<ExceptionBehavior<TRequest, TResponse>> logger)`
    - In `Handle`: wrap `await next()` in try/catch
    - Catch `FluentValidation.ValidationException`: log at Warning level with message `"Validation failed for {RequestName}"`, re-throw
    - Catch `UnauthorizedAccessException`: log at Warning level with message `"Authorization failed for {RequestName}"`, re-throw
    - Catch `Exception` (general): log at Error level with message `"Unhandled exception in {RequestName} {@Request}"`, re-throw
    - All exceptions are re-thrown -- the Function layer maps them to HTTP status codes

    **2. `LoggingBehavior.cs`**
    - Namespace: `WissensHub.Application.Common.Behaviors`
    - Primary constructor: `(ILogger<LoggingBehavior<TRequest, TResponse>> logger)`
    - In `Handle`: log at Information level `"Handling {RequestName}"` before calling `next()`
    - After `next()` completes, log at Information level `"Handled {RequestName} in {ElapsedMs}ms"`
    - Use `System.Diagnostics.Stopwatch.StartNew()` for timing

    **3. `AuthorizationBehavior.cs`**
    - Namespace: `WissensHub.Application.Common.Behaviors`
    - Primary constructor: `(ICurrentUser currentUser)`
    - Using statements: `using System.Reflection;`, `using WissensHub.Application.Common.Attributes;`, `using WissensHub.Application.Interfaces;`
    - In `Handle`: get all `RequireGroupAttribute` from `typeof(TRequest).GetCustomAttributes<RequireGroupAttribute>()`
    - For each attribute, check `currentUser.IsInGroup(attr.GroupName)`
    - If any group check fails, throw `UnauthorizedAccessException($"User is not a member of required group: {attr.GroupName}")`
    - If no attributes present, pass through to `next()` without any check

    **4. `ValidationBehavior.cs`**
    - Namespace: `WissensHub.Application.Common.Behaviors`
    - Primary constructor: `(IEnumerable<IValidator<TRequest>> validators)`
    - Using: `using FluentValidation;`
    - In `Handle`: if `!validators.Any()`, call `next()` directly
    - Otherwise create `ValidationContext<TRequest>(request)`, run all validators with `Task.WhenAll(validators.Select(v => v.ValidateAsync(context, ct)))`
    - Collect all failures: `.SelectMany(r => r.Errors).Where(f => f is not null).ToList()`
    - If `failures.Count > 0`, throw `new ValidationException(failures)`
    - Otherwise call `next()`
  </action>
