         c. Expose `access_as_user` delegated scope (Type="Admin" for admin-only consent):

            - `Update-MgApplication -ApplicationId $app.Id -Api @{ Oauth2PermissionScopes = @(...) }`

    3. Add Microsoft.Identity.Web NuGet package to `api/src/WissensHub.Functions/WissensHub.Functions.csproj`:
       - `dotnet add api/src/WissensHub.Functions/WissensHub.Functions.csproj package Microsoft.Identity.Web --version 3.8.0`

    4. Create `api/src/WissensHub.Functions/Middleware/AuthenticationMiddleware.cs`:
       - Sealed internal class implementing `IFunctionsWorkerMiddleware`
       - Constructor injects `IOptions<MicrosoftIdentityOptions>` (from Microsoft.Identity.Web) for tenant/client config
       - `Invoke(FunctionContext context, FunctionExecutionDelegate next)`:
         a. Check if HTTP trigger: `context.FunctionDefinition.InputBindings.Values.Any(b => b.Type == "httpTrigger")`
         b. If not HTTP trigger, call next(context) and return
         c. Get HTTP request data: `await context.GetHttpRequestDataAsync()`
         d. Skip auth for health endpoint: check if function name is "Health" -- if so, call next(context) and return. This keeps /api/health open during development.
         e. Extract Authorization header -- look for "Bearer " prefix
         f. If missing or invalid: create 401 Unauthorized response via `httpReqData.CreateResponse(HttpStatusCode.Unauthorized)`, set `context.GetInvocationResult().Value = response`, return
         g. Validate token using `JsonWebTokenHandler` with `TokenValidationParameters`:
            - ValidAudience: from config (api://{clientId})
            - ValidIssuer: `https://login.microsoftonline.com/{tenantId}/v2.0`
            - IssuerSigningKeys: fetched from OpenID Connect metadata endpoint `https://login.microsoftonline.com/{tenantId}/v2.0/.well-known/openid-configuration`
            - Use `ConfigurationManager<OpenIdConnectConfiguration>` for automatic key rotation
            - ValidateLifetime: true
         h. On validation success: store validated claims in `context.Items["User"]` for downstream functions to access, then call next(context)
         i. On validation failure: return 401 with error message
       - Use `Microsoft.IdentityModel.JsonWebTokens.JsonWebTokenHandler` (newer, recommended over JwtSecurityTokenHandler)
       - Cache the ConfigurationManager instance as a static field (it handles its own refresh)

    5. Update `api/src/WissensHub.Functions/Program.cs`:
       - Add `using Microsoft.Identity.Web;` and related usings
       - Before `builder.Build().Run()`, add:
         a. Configure authentication options from config section "AzureAd":
            `builder.Services.Configure<MicrosoftIdentityOptions>(builder.Configuration.GetSection("AzureAd"));`
         b. Register the middleware:
            `builder.UseMiddleware<AuthenticationMiddleware>();`
       - Keep existing EF Core DbContext registration unchanged

    6. Create or update `api/src/WissensHub.Functions/local.settings.json`:
