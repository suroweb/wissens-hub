**storage.bicep** -- Storage Account for Azure Functions runtime:
- Resource: `Microsoft.Storage/storageAccounts@2023-05-01`
- Parameters: `environment` (string), `location` (string)
- Name: `wh${environment}st` (storage accounts cannot have hyphens, max 24 chars)
- SKU: `Standard_LRS`
- Kind: `StorageV2`
- Properties: `allowBlobPublicAccess: false`, `supportsHttpsTrafficOnly: true`, `minimumTlsVersion: 'TLS1_2'`
- Output: `storageAccountName` (string), `storageAccountConnectionString` (string, constructed from listKeys)

**app-insights.bicep** -- Log Analytics workspace + Application Insights:
- Resource 1: `Microsoft.OperationalInsights/workspaces@2023-09-01`
  - Name: `wh-${environment}-law`
  - SKU: `PerGB2018`
  - Properties: `retentionInDays: 30`
- Resource 2: `Microsoft.Insights/components@2020-02-02`
  - Name: `wh-${environment}-ai`
  - Kind: `web`
  - Properties: `Application_Type: 'web'`, `WorkspaceResourceId: logAnalyticsWorkspace.id`, `DisableIpMasking: false`, `DisableLocalAuth: false`

- Parameters: `environment` (string), `location` (string)

**sql.bicep** -- Azure SQL Server + Database + firewall rules:
- Resource 1: `Microsoft.Sql/servers@2023-08-01-preview`
  - Name: `wh-${environment}-sql`
  - Properties: `administratorLogin: 'whadmin'`, `administratorLoginPassword: sqlAdminPassword`, `version: '12.0'`, `minimalTlsVersion: '1.2'`
- Resource 2: `Microsoft.Sql/servers/databases@2023-08-01-preview`
  - Name: `WissensHub` (parent: sqlServer)
  - SKU: `{ name: 'Basic', tier: 'Basic' }` (portfolio project, minimal cost)

**key-vault.bicep** -- Key Vault + secrets + RBAC assignments (per D-03):
- Resource 1: `Microsoft.KeyVault/vaults@2023-07-01`
  - Name: `wh-${environment}-kv`
  - SKU: `{ family: 'A', name: 'standard' }`
  - Properties: `tenantId: subscription().tenantId`, `enableRbacAuthorization: true`
- Resource 2: Secret `SqlConnectionString` with value from `sqlConnectionString` parameter
- Resource 3: Secret `AppInsightsKey` with value from `appInsightsInstrumentationKey` parameter
- Resource 4: Secret `AppInsightsConnectionString` with value from `appInsightsConnectionString` parameter
- Resource 5: `Microsoft.Authorization/roleAssignments@2022-04-01`
  - Key Vault Secrets User role (`4633458b-17de-408a-b874-0445c86b69e6`) for `functionAppPrincipalId`
  - Scope: key vault resource
  - Principal type: `ServicePrincipal`
- Parameters: `environment` (string), `location` (string), `sqlConnectionString` (secure string), `appInsightsInstrumentationKey` (string), `appInsightsConnectionString` (string), `functionAppPrincipalId` (string)
- Outputs: `keyVaultName` (string), `keyVaultUri` (string)

**function-app.bicep** -- App Service Plan + Function App + managed identity (per D-03):
- Resource 1: `Microsoft.Web/serverfarms@2024-04-01`
  - Name: `wh-${environment}-asp`
  - SKU: `{ name: 'Y1', tier: 'Dynamic' }` (Linux Consumption plan per research recommendation)
  - Kind: `linux`
  - Properties: `reserved: true` (required for Linux)
- Resource 2: `Microsoft.Web/sites@2024-04-01`
  - Name: `wh-${environment}-func`
  - Kind: `functionapp,linux`
  - Identity: `{ type: 'SystemAssigned' }`
  - Properties:
    - `serverFarmId: appServicePlan.id`
    - `httpsOnly: true`
    - `siteConfig.linuxFxVersion: 'DOTNET-ISOLATED|10.0'`
    - `siteConfig.appSettings` array:
      - `FUNCTIONS_EXTENSION_VERSION` = `~4`
      - `FUNCTIONS_WORKER_RUNTIME` = `dotnet-isolated`
      - `AzureWebJobsStorage` = storage account connection string (parameter)
      - `APPLICATIONINSIGHTS_CONNECTION_STRING` = `@Microsoft.KeyVault(VaultName=${keyVaultName};SecretName=AppInsightsConnectionString)`
      - `ConnectionStrings__DefaultConnection` = `@Microsoft.KeyVault(VaultName=${keyVaultName};SecretName=SqlConnectionString)`
    - `siteConfig.cors.allowedOrigins`: `['https://*.sharepoint.com']`
- Parameters: `environment` (string), `location` (string), `storageAccountConnectionString` (secure string), `keyVaultName` (string)
- Outputs: `principalId` (string -- managed identity principal ID), `functionAppName` (string), `defaultHostName` (string)
- NOTE: Circular dependency resolution -- the function app is deployed first with Key Vault references in app settings. The key-vault module then grants the function app's principalId access. On first deploy, Key Vault references will fail until RBAC propagates (this is expected and self-resolves within minutes).

**infra/main.bicep** -- Orchestrator deploying all child modules:
- `targetScope = 'resourceGroup'`
- Parameters:
  - `environment` (string, @allowed(['dev', 'prod']))
  - `location` (string, default: `resourceGroup().location`)
  - `sqlAdminPassword` (@secure() string)
- Module deployment order with output chaining:
  1. `module storage 'modules/storage.bicep'` -- passes `environment`, `location`
  2. `module appInsights 'modules/app-insights.bicep'` -- passes `environment`, `location`
  3. `module sql 'modules/sql.bicep'` -- passes `environment`, `location`, `sqlAdminPassword`
  4. `module functionApp 'modules/function-app.bicep'` -- passes `environment`, `location`, `storageAccountConnectionString: storage.outputs.storageAccountConnectionString`, `keyVaultName: 'wh-${environment}-kv'` (use string interpolation to break circular dep -- Key Vault name is deterministic)
  5. `module keyVault 'modules/key-vault.bicep'` -- passes `environment`, `location`, `sqlConnectionString: sql.outputs.connectionString`, `appInsightsInstrumentationKey: appInsights.outputs.instrumentationKey`, `appInsightsConnectionString: appInsights.outputs.connectionString`, `functionAppPrincipalId: functionApp.outputs.principalId`
- Outputs:
  - `functionAppName` (string, from functionApp.outputs.functionAppName)
  - `functionAppHostName` (string, from functionApp.outputs.defaultHostName)
  - `sqlServerFqdn` (string, from sql.outputs.serverFqdn)
  - `keyVaultName` (string, from keyVault.outputs.keyVaultName)
